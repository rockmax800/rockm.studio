# 11 — AIEmployee & HR Model

> Layer 2 — Company Layer

## 1 — Purpose

Defines the AI employee abstraction that wraps agent roles with named identity, performance metrics, and HR management.

---

---

## 3 — HR Operations

### Hiring
- Employees are created through founder-approved hiring proposals
- Each employee maps to one AgentRole + one provider/model combination
- Names are generated via `EmployeeNamingService`

### Performance Evaluation
- Periodic evaluation via `evaluate-employees` edge function
- Updates reputation_score, success_rate, bug_rate based on recent runs
- Generates HR suggestions when performance degrades

### Termination
- Never automatic — always requires founder approval
- Replacement proposal generated by `EmployeeReplacementService`
- Executed only after approval through `candidate_proposals` table

## 5 — Safety Rules

- Never auto-fire employees
- Never auto-switch model mid-task
- All proposals go through approval endpoint
- Only apply changes on next hire cycle
